目的：统一团队的Git Commit日志标准， 便于后续代码review和快速查找问题

参考 angular 的commit，中文版本git-commit-stype-guide

Commit Message日志基本规范

<type>(<scope>): <subject>

type是必选项，scope可选项，subject是必选项，body和footer可选项。

Type<类型>

代表每次提交的类型，具体有如下：

类型 描述
feat 新增功能
fix 修复bug
improvement 对当前功能的改进
docs 仅仅修改了文档
style 不影响代码逻辑的修改(空格、格式等)
refactor 代码重构，既没新增功能也没修复bug
perf 优化相关，比如提升性能、访问速度、体验等等
test 测试用例，比如单元测试、集成测试等
chore 构建过程或辅助工具的变动
revert 回滚到上一个版本
build 影响构建系统或外部依赖项的更改(例如作用域:gulp、broccoli、npm)
ci 对CI配置文件和脚本的更改(示例范围:Travis、Circle、BrowserStack、SauceLabs)

scope

用来说明本次Commit的影响范围，即简要说明修改会涉及的部分，比如应用层、模型层等，可为空。

subject

本次Commit的简短描述

body

对本次Commit的详细描述，可分为多行，可为空

footer

分两部分

  • 关闭issue【针对开源项目】

如果当前commit是针对某个issue，则可以以关闭，例如 Close #123 * 不兼容变动

如果当前代码与上一个版本不兼容，则 Footer 部分以BREAKING CHANGE开头，后面是对变动的描述、以及变动理由和迁移方法

当然了，也有一些工具，比如Commitizen

安装使用Commitizen

  • javascript npm install -g commitizen npm init --yes commitizen init cz-conventional-changelog --save --save-exact 安装完毕，使用git cz代替git commit命令即可 全局安装 在根目录创建.czrc文件，安装插件且写入cz适配器配置 > npm install -g commitizen cz-conventional-changelog > > echo ‘{ “path”: “cz-conventional-changelog” }’ > .czrc
